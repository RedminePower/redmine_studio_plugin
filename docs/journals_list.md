# Journals List（コメント履歴）

チケット一覧にコメント履歴を表示する機能です。

## 概要

チケット一覧の表示オプションに「コメント履歴」を追加します。有効にすると、各チケットの行の下にコメント一覧がテーブル形式で表示され、レビューのやり取り（誰が何のコメントをしたか）を一覧で俯瞰できます。

「説明」や「最新のコメント」と同じブロックカラムとして動作するため、Redmine 標準のチケット一覧画面と `ref_issues` マクロの両方で使用できます。

### 主な機能

- チケット一覧に各チケットのコメント一覧を表示
- 各コメント時点のステータスと担当者を表示
- 各コメントの折りたたみ/展開（Wiki レンダリング対応）
- カラムヘッダーによるソート（#、更新者、更新日、ステータス、担当者、コメント）
- ヘッダー行のダブルクリックで詳細の展開/折りたたみ
- 右クリックコンテキストメニューによる一括展開/折りたたみ
- プライベートコメントの権限制御に対応

## 使い方

### チケット一覧画面

1. チケット一覧の「オプション」を開く
2. 「表示」セクションの「コメント履歴」にチェックを入れる
3. 「適用」をクリック

### ref_issues マクロ

Wiki やチケットの説明欄で、`journals_list` をカラムに指定します。

```
{{ref_issues(-p, id, subject, journals_list)}}
```

## 表示内容

### ヘッダー行

各コメントについて、以下の情報が1行で表示されます。

| 項目 | 内容 |
|------|------|
| # | コメント番号（クリックでチケット詳細画面の該当コメントにジャンプ） |
| 更新者 | コメントの投稿者名（クリックでユーザーページにジャンプ） |
| 更新日 | コメントの投稿日時 |
| ステータス | そのコメント時点でのチケットのステータス |
| 担当者 | そのコメント時点でのチケットの担当者（クリックでユーザーページにジャンプ） |
| コメント | コメントの1行目（プレーンテキスト、最大100文字） |

### ステータス・担当者の表示ルール

- 各コメント行には、そのコメントから次のコメントまでの間に最終的に確定したステータスと担当者が表示されます
- コメントの追加と同時にステータスや担当者が変更された場合は、変更後の値が表示されます
- コメントのないジャーナル（属性変更のみ）は行として表示されませんが、属性変更は前のコメント行の値に反映されます
- 最後のコメントには、現在のステータスと担当者が表示されます

### 詳細表示

「表示」ボタンのクリック、またはヘッダー行のダブルクリックで、コメントの全文が Wiki 記法でレンダリングされて表示されます。

- 展開時のコンテンツは AJAX で遅延ロードされるため、ページの表示速度に影響しません
- 一度展開したコンテンツはキャッシュされ、2回目以降はサーバーにリクエストを送りません
- ヘッダー行のダブルクリックで展開/折りたたみを切り替えられます
- 詳細部分のダブルクリックはテキスト選択として動作します（折りたたまれません）

### 表示ルール

- ノートが空のジャーナル（ステータス変更のみなど）は表示されません
- プライベートコメントは、閲覧権限がないユーザーには表示されません

## ソート

カラムヘッダー（#、更新者、更新日、ステータス、担当者、コメント）をクリックすると、その列でソートできます。

- クリックするたびに昇順/降順が切り替わります
- ソートアイコン（∧/∨）でソート方向を確認できます
- 展開中のコメントはソート後も展開状態が維持されます

## 右クリックメニュー

コメント履歴のコメント行を右クリックすると、以下のメニューが表示されます。

| メニュー項目 | 動作 |
|-------------|------|
| 詳細を表示 | 右クリックしたコメントの詳細を展開（折りたたみ時のみ表示） |
| 詳細を隠す | 右クリックしたコメントの詳細を折りたたみ（展開時のみ表示） |
| すべての詳細を表示 | 同じチケットのすべてのコメントを一括展開 |
| すべての詳細を隠す | 同じチケットのすべてのコメントを一括折りたたみ |

展開されたコメントの詳細部分（Wiki レンダリング領域）を右クリックしても、同じメニューが表示されます。
