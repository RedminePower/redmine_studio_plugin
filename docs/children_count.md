# Children Count（子チケット数）

チケット一覧に「子チケット数」カラムを追加する機能です。

## 概要

チケット一覧に各チケットの直下の子チケット数を表示します。子チケット数の多いチケットは、レビュー指摘やサブタスクが多いことを示唆します。例えばトラッカー「レビュー開催」のチケットを一覧表示すれば、指摘の多いレビューを即座に把握できます。

### 主な機能

- チケット一覧に子チケット数をインラインカラムとして表示
- 子チケット数でソート可能（デフォルトは降順）
- カウントをクリックすると、当該チケットを親に持つチケット一覧画面に遷移
- カーソルを合わせると子チケットの ID と件名をツールチップで表示

## 使い方

### チケット一覧画面

1. チケット一覧の「オプション」を開く
2. 「利用できる項目」から「子チケット数」を「選択された項目」に追加
3. 「適用」をクリック

### 子チケット一覧への遷移

カウントが 1 以上のチケットは、数字部分がリンクになっています。クリックすると、当該チケットを親に持つチケットの一覧画面（クロスプロジェクト検索）に遷移します。子チケットが別プロジェクトに移っている場合も、まとめて確認できます。

カウントが 0 のチケットはリンクになりません。

## カウントの仕様

- **直下の子チケットのみ** をカウント（孫以降は含めない）
- 現在ログイン中のユーザーが閲覧可能な子チケットだけをカウント（権限のないプロジェクトの子チケットはカウントに含まれません）

## ツールチップ

子チケット数にカーソルを合わせると、子チケットの ID と件名がツールチップで表示されます。

**表示例（子チケット数 = 3）:**
```
#1234 仕様レビューでの指摘事項1
#1235 仕様レビューでの指摘事項2
#1236 仕様レビューでの指摘事項3
```

- 最大 10 件まで表示されます。10 件を超える場合は、最後に `...他 N 件` が表示されます
- 件名は 30 文字を超える場合、末尾に `...` を付けて省略されます
- ログイン中のユーザーが閲覧可能な子チケットだけが表示されます
