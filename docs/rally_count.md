# Rally Count（ラリー回数）

チケット一覧に「ラリー回数」カラムを追加する機能です。

## 概要

チケットの担当者が何回切り替えられたかを「ラリー回数」として表示します。ラリー回数が多いチケットは、要件の認識齟齬やレビュー指摘の多さを示唆する指標として活用できます。

### 主な機能

- チケット一覧にラリー回数をインラインカラムとして表示
- ラリー回数でソート可能（デフォルトは降順）
- カーソルを合わせると担当者の変更履歴をツールチップで表示

## 使い方

### チケット一覧画面

1. チケット一覧の「オプション」を開く
2. 「利用できる項目」から「ラリー回数」を「選択された項目」に追加
3. 「適用」をクリック

## ラリー回数のカウントルール

- 担当者が変更されるたびに +1
- 担当者なし → ユーザー = +1（初回設定も含む）
- ユーザー → 担当者なし = +1（クリアも含む）
- チケット作成時の担当者設定はラリーに含まない（初期状態）
- 一度も変更がない場合 = 0

## ツールチップ

ラリー回数にカーソルを合わせると、担当者の変更履歴がツールチップで表示されます。

**表示例（ラリー回数 = 3）:**
```
田中 太郎
 - 佐藤 花子
 - 田中 太郎
 - （担当者なし）
```

- 1行目（インデントなし）: チケット作成時の初期担当者
- 2行目以降（` - ` 付き）: 担当者の変更先
- 担当者が設定されていない場合は「（担当者なし）」と表示
- 削除済みユーザーは「（担当者なし）」と表示
